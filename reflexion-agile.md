​a) ¿Qué diferencia hay entre Scrum y Kanban? ¿Cuál te parece más útil para un proyecto individual?
​Scrum se organiza en ciclos de tiempo fijos llamados Sprints, mientras que Kanban es un flujo continuo que se visualiza en un tablero. Para un proyecto individual  me parece más útil Kanban porque permite ver el progreso de las  tareas sin depender de reuniones.
​b) ¿Qué es un MVP (Producto Mínimo Viable) y cómo lo aplicarías si te piden hacer un sitio web para un negocio local?
​Un MVP es la versión más básica de un producto que ya funciona y sirve para que el cliente lo use. Si tuviera que hacer un sitio para un negocio, mi MVP sería una página con el nombre del local, fotos de los productos disponibles y agregar una seccion para pedidos .
